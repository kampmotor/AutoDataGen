R2S 的本质：为什么需要"回到仿真"

传统 Sim-to-Real 的核心痛点是域差距（domain gap）——仿真中训练的策略在真实世界中表现不佳。R2S2R 的核心洞察是：与其让策略适应仿真，不如让仿真适应真实世界。

Real
真实场景 → Sim
数字孪生构建 → Train
仿真中训练/生成数据 → Real
部署到真实世界

R2S 的价值：真实世界数据采集成本 $100+/小时，而 R2S 允许从几分钟的扫描/照片构建数字孪生，然后在仿真中无限生成训练数据。这不是"替代"真实数据，而是"放大"少量真实数据的价值。

第一层：3DGS + 物理引擎（主流路线）

用 3D Gaussian Splatting 从真实场景的多视角照片/视频重建高保真 3D 场景，再对接物理仿真引擎（Isaac Sim、MuJoCo 等）。这是当前 R2S 的主流技术路线。

2024 · CoRL

SplatSim

核心：用 3DGS 替代传统 mesh 作为渲染原语，保留仿真器的物理引擎。零样本 sim2real 成功率 86.25%（vs 真实数据训练 97.5%）。

输入：多视角 RGB 图像 → 3DGS 重建 → 替换仿真器中的 mesh 渲染

2024

RoboGSim

核心：3DGS 重建 + Isaac Sim 数字孪生构建 + 布局对齐模块。支持演示合成和闭环策略评估。

输入：多视角 RGB + 机械臂 MDH 参数 → Isaac Sim 可交互场景

2024

RL-GSBridge

核心：Mesh-based 3DGS + 软绑定约束，减少非结构化物体的渲染伪影。

成果：抓取和放置任务的零样本 sim2real 迁移

2025

Re3Sim

核心：MVS 多视图立体几何重建 + 高斯渲染 + PhysX 物理后端。零样本平均成功率 58%。

特点：同时解决几何 gap 和视觉 gap，生成大规模仿真数据集

2025

VR-Robo

核心：3DGS + 基础模型先验 → 四足机器人导航的数字孪生。

特点：GS-mesh 混合表示 + 坐标对齐 + 智能体-物体随机化 + 遮挡感知场景组合

第一层特征：解决"视觉对齐"——让仿真渲染看起来像真实世界。但物体通常是静态的，缺乏物理交互能力（不能推、拉、抓）。

第二层：3DGS + 物理属性推断（可交互性）

第一层的 3DGS 重建只能渲染静态场景。前沿工作用多模态大模型（MLLM）自动推断物体的物理属性和运动学结构，将静态场景变为可交互的"游乐场"。

2025

RoboSimGS

架构：3DGS（静态背景）+ Mesh（可交互物体）+ MLLM 自动推断密度、刚度、铰接结构（铰链、抽屉滑轨）

突破：从多视角图片直接推断物体物理属性 → 零样本 sim2real

2026

ArtiTwinSplat

架构：从 RGB-D 视频无监督发现铰接结构 → SAM2 部件跟踪 → 关节估计 → 高斯优化

突破：直接导出 URDF → Isaac Sim 可用，无需 CAD 模型或人工标注

2025

TwinAligner

架构：SDF 重建 + 3DGS 可编辑渲染 + 物理对齐模块（同时优化视觉和动力学）

突破：闭环同步——从真实机器人交互数据推断刚体物理参数

2025

Splatting Physical Scenes

架构：SplatMesh（3DGS + mesh 混合）+ 可微物理（MuJoCo MJX）→ 端到端优化

突破：视觉差异梯度直接传播到几何、机器人姿态、相机参数

第二层特征：解决"物理对齐"——不仅看起来像真实世界，还要动起来像真实世界。MLLM/基础模型成为推断物理属性的关键桥梁。

第三层：生成式 Real2Sim（跳过重建）

不再从真实场景"重建"，而是用生成模型直接从图片/文本生成仿真环境。这是最具扩展性的路线——一张互联网图片就能变为可训练的仿真场景。

2024

URDFormer

流程：单张 RGB 图 → Transformer 预测 URDF → 加载到仿真器

规模：支持 5 类物体（橱柜、烤箱、洗碗机、冰箱、洗衣机），从互联网图片直接生成仿真环境

成果：零样本 real2sim2real，开/关柜门成功率 78%

2026

WorldComposer

流程：全景图 → 生成式 3DGS → 高保真仿真场景 + 语义/几何编辑

创新："数字表亲"（Digital Cousins）—— 在真实场景基础上生成多样化变体，而非精确复制

规模：多房间拼接 → 大规模一致环境 → 长时域导航

2026

ComSim

流程：经典仿真（动作精确）+ 神经仿真（DiT 视频生成，视觉逼真）→ 组合生成伪真实数据

创新：仅 10 条真实演示即可 bootstrap 大规模训练数据

成果：显著缩小 sim2real 域差距，提升策略成功率

第三层特征：解决"规模"——不再受限于真实场景扫描，可以从互联网图片/文本无限生成仿真训练环境。这是通向"无限数据"的关键路径。

第四层：动态数字孪生（实时同步）

最激进的 R2S 路线：仿真器与真实世界持续同步，策略始终在仿真中执行，真实机器人仅作为仿真状态的"物理执行器"。

2025

real-is-sim

范式转换：策略永远在仿真中运行，真实机器人跟随仿真关节状态。将 sim2real 的责任从策略转移到数字孪生的同步机制。

同步频率：60Hz 实时同步

核心：Embodied Gaussian 模拟器 + 真实世界持续校正

2026

ExoGS

创新：低成本外骨骼（~$400）+ 3DGS 数字化 → 无需真实机器人即可采集训练数据

流程：人类佩戴外骨骼操作 → 3DGS 重建场景 → 几何感知数据增强 → 策略训练

成果：数据采集成本大幅降低，策略性能超过传统遥操作

第四层特征：解决"部署"——不再区分"仿真训练"和"真实部署"，仿真器成为策略的永久运行环境。这是 R2S 的终极形态，但对同步精度要求极高。

核心发现：光保真不够

关键结论（2026, OpenReview）：仅靠光保真重建不足以支撑 VLA 策略的闭环部署。在接触丰富操作中，微小感知误差会随时间累积，导致轨迹发散。

一项系统性评估发现，即使数字孪生在视觉上高度逼真，VLA 策略在闭环执行中仍会逐渐偏离真实世界行为。这揭示了 R2S 的核心挑战：

视觉 Gap

外观对齐

3DGS 已能生成照片级渲染。但光照变化、反射、透明材质仍是挑战。Cosmos Transfer 等工具正在弥补。

物理 Gap

动力学对齐

物体质量、摩擦、刚度、接触力等物理参数难以从视觉推断。TwinAligner 和 RoboSimGS 正在突破。

动态 Gap

可变形物体

绳子、布料、液体的仿真极其困难。SoftMimicGen 和 Real2Sim-Eval 正在探索。

与 AutoDataGen 的融合路径

AutoDataGen 目前是纯 Sim 路线——从任务描述直接在仿真中生成数据。如果要走向 R2S2R，最自然的演进路径是：

真实场景
手机/RGB-D 扫描 → 3DGS/NeRF 重建
数字孪生构建 → Isaac Lab
加载数字孪生 → AutoDataGen
LLM 分解 + 运动规划 → 训练数据
→ 策略训练 → 部署

具体融合方案

短期

方案一：场景加载适配

AutoDataGen 的 load_env() 接口已支持加载 Isaac Lab 环境。只需增加从 URDF/USD 文件加载数字孪生的能力，即可将 R2S 重建的场景接入 AutoDataGen 管线。

中期

方案二：感知增强分解

当前 LLM 分解器依赖文本描述。增加视觉感知模块（VLM），从数字孪生的渲染图像中提取场景信息，实现基于视觉的任务分解。

长期

方案三：闭环数据飞轮

策略在数字孪生中执行 → 失败案例反馈 → 针对性生成新数据 → 策略改进。结合 R2S 的真实场景感知和 AutoDataGen 的自主数据生成能力。

核心价值：AutoDataGen 的 LLM 分解 + 运动规划能力 + R2S 的真实场景感知 = 从真实世界照片直接生成训练数据。这是"Text2Sim2Real"到"Photo2Sim2Real"的跨越。
